# 07 前端统一视觉规范

## 设计目标

三端前端统一采用“清爽蓝色校园风”。学生端和教师端更接近用户提供的参考图：蓝天校园插画、白色圆角卡片、蓝色主按钮、柔和阴影、轻量图标。管理员 Web 端保持后台系统的工作效率，不做大面积插画堆叠，但使用同一套色彩、圆角、组件状态和图标语言。

视觉关键词：

- 明亮、干净、校园感。
- 蓝色主视觉，白色卡片承载内容。
- 大圆角、柔和阴影、低边框对比。
- 任务状态清晰，操作按钮明确。
- 学生端亲和，教师端高效，管理员端稳重。

## 色彩系统

主色：

- `Primary Blue`: `#1677FF`
- `Primary Blue Light`: `#EAF4FF`
- `Primary Blue Deep`: `#0066E6`
- `Sky Blue`: `#2EA8FF`

辅助色：

- `Success Green`: `#20C55A`
- `Warning Orange`: `#FF9F1A`
- `Alert Purple`: `#7C5CFF`
- `Danger Red`: `#F04438`

中性色：

- `Text Primary`: `#101828`
- `Text Secondary`: `#667085`
- `Text Muted`: `#98A2B3`
- `Border Light`: `#D8E8FF`
- `Page Background`: `#F3F8FF`
- `Card Background`: `#FFFFFF`

移动端可使用蓝色渐变背景：

```text
linear-gradient(180deg, #0F8BFF 0%, #38B6FF 48%, #F3F8FF 100%)
```

管理员端页面背景使用浅蓝灰，不使用全屏强渐变：

```text
#F3F8FF 或 #F6FAFF
```

## 圆角与阴影

移动端：

- 页面大卡片：`28px` 至 `32px`。
- 输入框：`16px` 至 `18px`。
- 功能入口图标底座：`18px` 至 `22px`。
- 主按钮：`18px` 至 `22px`。

管理员 Web 端：

- 页面卡片：`10px` 至 `12px`。
- 表格筛选区：`10px`。
- 按钮：优先使用 Element Plus 默认圆角或轻微增强，不使用移动端超大圆角。

阴影使用轻柔蓝色阴影：

```text
0 10px 30px rgba(22, 119, 255, 0.10)
0 4px 14px rgba(16, 24, 40, 0.06)
```

## 移动端页面结构

### 登录/激活页

参考图中的结构：

- 顶部蓝色校园插画背景。
- 左上放产品名“考勤助手”或系统名。
- 副标题使用“智慧考勤，轻松校园生活”。
- 主体为白色大圆角卡片。
- 登录和激活使用 Tab。
- 输入框左侧使用蓝色图标。
- 主按钮使用蓝色渐变。
- 协议链接使用主蓝色。

登录字段：

- 学号/手机号。
- 密码。
- 验证码，可选。

首次激活字段：

- 姓名。
- 学号。
- 手机号。
- 验证码。
- 设置密码。
- 确认密码。
- 用户协议和隐私政策勾选。

### 首页

学生端和教师端首页都采用：

- 顶部蓝色校园插画区域。
- 问候语和身份信息。
- 下方白色功能卡片。
- 关键功能入口采用彩色图标块。
- 内容卡片纵向排列：通知、今日任务、课程/班级、每日一句等。

学生端功能入口建议：

- 签到打卡。
- 考勤记录。
- 统计分析。
- 请假申请。

教师端功能入口建议：

- 创建任务。
- 今日任务。
- 异常处理。
- 班级数据。

### 底部导航

学生端：

- 首页。
- 打卡任务。
- 消息。
- 我的。

教师端：

- 首页。
- 任务。
- 班级。
- 我的。

如果采用中间凸起主按钮，只在学生端用于“打卡”，教师端不建议使用凸起按钮，避免与“创建任务”产生歧义。

## 管理员 Web 端统一方式

管理员端不要直接做成移动端风格的大插画页面。建议采用“蓝白后台风”：

- 左侧导航使用深浅蓝或白底蓝色激活态。
- 顶部栏使用白底，保留学校/学院切换、搜索、消息和账号。
- 工作台可使用浅蓝渐变头部，但高度控制在 160px 以内。
- 数据卡片使用白底、轻阴影、主蓝色图标。
- 表格、筛选、弹窗沿用 Element Plus，但覆盖主色为 `#1677FF`。

管理员端可少量使用校园插画：

- 登录页背景。
- 工作台欢迎区右侧插画。
- 空状态插画。

不要在表格密集页面使用大面积插画，避免影响管理效率。

## 组件规范

### 按钮

- 主按钮：蓝色渐变或纯 `#1677FF`。
- 次按钮：白底蓝字蓝边。
- 危险按钮：红色，只用于删除、禁用、驳回等高风险动作。
- 移动端按钮高度建议 `48px` 至 `54px`。
- 管理端按钮高度按 Element Plus 中等尺寸。

### 输入框

- 移动端输入框使用白底、浅蓝边框、左侧图标。
- 聚焦边框使用主蓝色。
- 错误状态使用红色边框和短提示。

### 卡片

- 移动端卡片用于功能入口、通知、任务、课程、每日一句。
- 管理端卡片用于数据指标、筛选区、表格外框、详情模块。
- 不要把卡片嵌套成多层卡片。

### 图标

图标风格统一为圆润、填充或线性+填充混合。移动端功能入口可用高饱和渐变图标块：

- 蓝：签到、定位、任务。
- 绿：记录、通过、完成。
- 橙：统计、提醒、待处理。
- 紫：消息、通知、申诉。

管理员端使用更克制的线性图标，颜色跟随主色和状态色。

## 状态表达

统一状态色：

- 正常/已完成：绿色。
- 待完成/进行中：蓝色。
- 待审核/待处理：橙色。
- 异常/驳回：红色。
- 已归档/已结束：灰色。

任务卡片、表格标签、小程序消息都使用同一套状态文本：

- 未开始。
- 进行中。
- 已完成。
- 已结束。
- 待审核。
- 异常。
- 已驳回。

## 实施要求

第一阶段应抽出基础设计变量：

- uni-app：统一 SCSS 变量或 CSS 变量，放在 `src/styles/tokens.scss`。
- 管理端：统一 Element Plus theme tokens，放在 `src/styles/theme.scss`。

两端应共用同一份命名规则：

- `primary`
- `success`
- `warning`
- `danger`
- `muted`
- `card`
- `page-bg`

前端实现时先做登录页和首页作为视觉基准页，再推广到任务、消息、我的、管理端工作台和表格页面。
